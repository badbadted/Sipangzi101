# UIUX 建議：各空間需求清單（BriefingView）

## 現況分析

目前的排版為「一空間一列」水平展開：

```
┌─────────┬──────────────┬──────────┬──────────┬────────┐
│ 空間名稱 │   需求項目    │   家俱   │   裝潢    │  圖片  │
│ + 優先度 │              │          │          │        │
└─────────┴──────────────┴──────────┴──────────┴────────┘
```

### 現有優點
- 樓層分組 + 顏色區分，結構清楚
- 優先度標籤一目了然
- Header 有統計摘要（高/中/低數量）

### 現有問題
1. **水平溢出風險**：家俱 + 裝潢 + 圖片三欄固定寬度（w-52 + w-52 + w-36 = 560px），加上左側 160px + 中間需求欄，總寬容易超出容器
2. **資訊密度不均**：有些空間只有名稱沒有內容，卻佔一整列；有些空間家俱很多卻擠在 w-52 內
3. **缺少完成度指示**：無法快速辨識哪些空間「需求已齊全」vs「尚待補充」
4. **無法快速定位**：空間多時缺少錨點導航或搜尋
5. **手機版未處理**：flex 水平排列在小螢幕會破版

---

## 建議改版方向

### 方案 A：表格式總覽（推薦）

最適合 PC 端「一目了然」的需求，類似 spreadsheet 風格：

```
┌──────┬────┬─────┬────────────────┬──────────┬──────────┬──────┐
│ 樓層 │空間│優先度│    需求摘要     │ 家俱數量  │ 裝潢數量  │ 圖片 │
├──────┼────┼─────┼────────────────┼──────────┼──────────┼──────┤
│ 1F   │客廳│ 高  │ 採光要好、開放.. │ 沙發 +2   │ 油漆 +1  │ 3張  │
│ 1F   │廚房│ 中  │ 中島設計...     │ —        │ 磁磚 +2  │ 1張  │
│ 2F   │主臥│ 高  │ 衣櫃收納...     │ 床架 +3   │ 壁紙 +1  │ —   │
│ 2F   │書房│ 低  │ 尚未填寫        │ —        │ —       │ —   │
└──────┴────┴─────┴────────────────┴──────────┴──────────┴──────┘
                                         ↕ 點擊列可展開明細
```

**特點：**
- Sticky header，捲動時欄位標題固定
- 點擊列展開該空間完整明細（家俱清單、裝潢清單、參考圖片大圖）
- 需求摘要顯示前 N 字 + 省略號
- 家俱/裝潢欄顯示「首項名稱 +N」，滑鼠 hover 顯示 tooltip 完整清單
- 完成度圓餅圖或進度條（需求/家俱/裝潢/圖片 各項是否已填）

### 方案 B：Dashboard 卡片 + 側邊明細面板

```
┌──────────────────────────────┬─────────────────────────┐
│ ┌──────┐ ┌──────┐ ┌──────┐  │                         │
│ │ 客廳 │ │ 廚房 │ │ 餐廳 │  │    ← 選取的空間明細 →    │
│ │ 1F高 │ │ 1F中 │ │ 1F中 │  │                         │
│ │ ●●●○ │ │ ●●○○ │ │ ●○○○ │  │  需求：                 │
│ └──────┘ └──────┘ └──────┘  │  - 採光要好              │
│ ┌──────┐ ┌──────┐           │  - 開放式格局            │
│ │ 主臥 │ │ 書房 │           │                         │
│ │ 2F高 │ │ 2F低 │           │  家俱：沙發、茶几、電視櫃 │
│ │ ●●●● │ │ ○○○○ │           │  裝潢：油漆              │
│ └──────┘ └──────┘           │  [參考圖片 grid]         │
└──────────────────────────────┴─────────────────────────┘
```

**特點：**
- 左側為空間縮圖卡片（小卡），●○○○ 表示完成度
- 點擊卡片後右側面板展示完整明細
- 卡片可拖曳排序
- 適合空間數量較多（>10）的專案

### 方案 C：保留現有一列式，加強互動

維持目前一列一空間的結構，但加入以下改進：

```
[搜尋空間] [篩選：全部 ▾] [排序：樓層 ▾]      [展開全部 / 收合全部]
─────────────────────────────────────────────────────────────────

▸ 1F ─────────────────────────────────── 3 個空間

┌─────────┬──────────────────────────────────────────────────────┐
│         │ 需求(3)  採光要好 · 開放式格局 · 預留投影幕            │
│  客廳   ├──────────────────────────────────────────────────────┤
│  高優先  │ 家俱(3) 沙發·茶几·電視櫃  裝潢(1) 油漆  圖片(3)      │
└─────────┴──────────────────────────────────────────────────────┘

┌─────────┬──────────────────────────────────────────────────────┐
│  廚房   │ 需求(1)  中島設計                                     │
│  中優先  │ 家俱(0) —  裝潢(2) 磁磚·櫥櫃  圖片(1)                │
└─────────┴──────────────────────────────────────────────────────┘
```

**改進點：**
- 需求改為一行式摘要（用 · 分隔），不再逐條列出
- 家俱/裝潢/圖片合併為同一行底部欄
- 加入搜尋 + 篩選 + 排序工具列
- 展開/收合控制

## 通用建議（不論選哪個方案）

### 1. 完成度指示器
每個空間顯示填寫進度，快速辨識哪些空間還需補充：

```
客廳 ████░ 80%    （需求 ✓ 家俱 ✓ 裝潢 ✓ 圖片 ✗）
書房 █░░░░ 20%    （需求 ✓ 家俱 ✗ 裝潢 ✗ 圖片 ✗）
```

### 2. 篩選與排序工具列
```
[全部樓層 ▾] [全部優先度 ▾] [排序：樓層 ▾] [只看未完成 ☐]
```

### 3. 展開/收合互動
- Header 旁加「展開全部 / 收合全部」按鈕
- 每列可點擊展開詳細內容（完整需求列表、家俱大圖、裝潢明細）
- 收合時只顯示摘要行

### 4. 空狀態優化
空間內容為空時顯示引導：
```
┌─────────┬──────────────────────────────────────────┐
│  書房   │  ⚠ 尚未填寫任何需求，點此開始編輯 →       │
│  低優先  │                                          │
└─────────┴──────────────────────────────────────────┘
```

### 5. 響應式設計
- **PC (>1024px)**：水平一列式，所有欄位展開
- **平板 (768-1024px)**：左側空間名 + 右側堆疊
- **手機 (<768px)**：改為垂直卡片堆疊

### 6. 匯出功能
Header 加入「匯出 PDF / 列印」按鈕，方便與客戶或工班分享需求清單。

## 推薦優先順序

| 優先度 | 項目 | 預估影響 |
|--------|------|---------|
| P0 | 響應式設計（手機不破版） | 解決破版問題 |
| P0 | 展開/收合互動 | 資訊量大時可控制密度 |
| P1 | 完成度指示器 | 快速辨識未完成空間 |
| P1 | 方案 C 摘要式排版改進 | 最小改動最大效益 |
| P2 | 篩選排序工具列 | 空間多時實用 |
| P2 | 空狀態引導 | 提升新手體驗 |
| P3 | 方案 A 表格式總覽 | 大幅改版，適合下階段 |
| P3 | 匯出 PDF | 商業功能 |
